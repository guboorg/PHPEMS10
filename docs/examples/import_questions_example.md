# 导入试题 CSV 示例说明

`import_questions_example.csv` 是根据 `app/exam/cls/exam.cls.php` 中 `importQuestionBat()` 的读取顺序整理的可导入示例文件。系统逐行读取 CSV，不跳过表头，因此 CSV 文件本身不包含表头。

## 字段顺序

| 序号 | 字段 | 说明 |
| --- | --- | --- |
| 1 | questiontype | 题型 ID。默认数据：1 单选题、2 多选题、3 判断题、4 定值填空、5 填空题、6 问答题、7 题冒题。 |
| 2 | question | 题干。支持 HTML。 |
| 3 | questionselect | 备选项。客观选择题可使用 HTML，例如 `<p>A:选项</p>`；主观题或填空题可留空。 |
| 4 | questionselectnumber | 备选项数量。无选项题填 `0`；示例覆盖了 2、3、4、5、6、7、8 个选项。 |
| 5 | questionanswer | 标准答案。多选题可写 `A,B`。 |
| 6 | questiondescribe | 解析。 |
| 7 | knowsid | 知识点 ID，多个 ID 用英文逗号分隔；如果上传时已指定知识点，系统会优先使用上传参数中的知识点。 |
| 8 | questionlevel | 难度，常用 `1`、`2`、`3`。 |
| 9 | isqr | 是否题冒题/组合题：`1` 是，`0` 否。 |
| 10 | istitle | 当 `isqr=1` 时，`1` 表示组合题题干行，`0` 表示组合题子题行。 |

## 注意事项

- 导入代码使用 `fgetcsv()` 逐行读取，且没有表头跳过逻辑，所以请不要在正式导入文件第一行放表头。
- 导入代码会按 `GBK` 到 `UTF-8` 转码处理中文内容。如果 CSV 中包含中文，建议另存为 `GBK/ANSI` 编码后再导入；本示例 CSV 使用 ASCII 内容，避免编码差异导致乱码。
- 默认题型中没有“不定项选”题型，但系统题型管理支持答题方式 `questchoice=3`。如需导入不定项选，请先在题型管理中新增该题型，再把 CSV 第一列改成新增题型 ID，答案格式可参考多选题。
